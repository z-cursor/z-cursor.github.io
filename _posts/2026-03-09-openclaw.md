# Mac 安装教程
> Mac 上装 OpenClaw 是最省心的, 基本上一行命令搞定

## 第一步:打开终端

按`Command + 空格`, 输入`Terminal`, 回车, 终端就出来了
如果你用 iTerm2 之类的第三方终端也完全 OK

## 第二步:一键安装

在终端里粘贴这行命令, 回车:

`curl -fsSL https://openclaw.ai/install.sh | bash` 

这个脚本会自动帮你:

* 检测系统环境
* 安装 Node.js（如果你没装的话）
* 下载 OpenClaw
* 配好环境变量

整个过程大概 2-5 分钟, 泡杯咖啡的功夫就好了

## 第三步:检查安装
终端输入`openclaw --version`并回车

如果终端输出了版本号（比如 2026.3.7）, 那说明openclaw成功安装

## 第四步:初始化配置

这一步是重头戏, 运行配置向导:

终端输入`openclaw onboard`并回车

向导会一步步问你以下内容:

* 接受风险提示 —— 选 Yes, 放心往下走
* 选择启动模式 —— 推荐选 QuickStart, 省事
* 选择 AI 模型

> ![](https://raw.githubusercontent.com/z-cursor/blog-pic/main/Post/openclaw2.png)选择模型的界面

新手怕麻烦的话, 直接选 qwen（通义千问）就行！ 选了之后终端内弹出来一个链接, 如下图红框中的链接, 把链接复制下来, 粘贴到浏览器中, 就到Qwen的登录页面, 授权一下就完事了, 不需要自己去申请什么 API Key这是最省心的方式, 适合想赶紧用起来的同学

> ![](https://raw.githubusercontent.com/z-cursor/blog-pic/main/Post/openclaw3.png)openclaw终端中需要复制的Qwen链接

不过要注意:Qwen 免费额度每天问答次数有限, 超了之后机器人会提示你如果你用得比较频繁, 还是建议后面配一个自己的 API Key

4. 输入 API Key（选了 Qwen 的可以跳过这步） —— 去对应模型的官网申请, 复制粘贴进去就行
5. 选择聊天工具 —— 飞书、Telegram 或者先选 None 跳过（后面可以在图形化界面中配置）
6. Gateway 端口 —— 默认 18789, 直接回车跳过
7. 选择 Skills —— 用空格键勾选你需要的技能包, 不确定的话直接跳过
8. 启用 Hooks —— 建议启用, 用于内容引导、日志和会话记录

配置完成后, OpenClaw 会自动启动 Gateway 服务

## 第五步:验证运行

终端输入`openclaw channels status`并回车

看到`Gateway reachable`. 就说明一切正常

打开浏览器访问 `http://127.0.0.1:18789/chat`, 就能看到 Web 聊天界面, 聊天界面如下:

![](https://raw.githubusercontent.com/z-cursor/blog-pic/main/Post/openclaw5.png)

日常使用命令, 所有的命令都在终端中执行, 也可以需要时随用随问AI:

- 启动openaclaw:
`openclaw gateway start`

- 开机自启（推荐）:
`openclaw gateway enable`

- 停止服务:
`openclaw gateway stop`

- 更新 OpenClaw:
`openclaw update `

# Windows 安装教程

> Windows 上有两种方式, 方案一是windows版本, 方案二是wsl版本, 都可以装 OpenClaw, 按自己情况选

## 方案一:PowerShell 原生安装

### 第一步:装 Node.js

去 [Node.js 官网](https://nodejs.org/zh-cn/download) 下载 22.x LTS 版本的 Windows 安装包, 具体下载界面如下图，一路 Next 装的时候记得勾上「自动安装必要的工具」

![](https://raw.githubusercontent.com/z-cursor/blog-pic/main/Post/nodejs.png)

### 第二步:确认 Node.js 装好了

在搜索框中输入powershell, 下图即是powershell:
![](https://raw.githubusercontent.com/z-cursor/blog-pic/main/Post/powershell2.png)

打开PowerShell后分别输入`node -v`和`npm -v` ,如都能输出版本号即证明node.js安装成功

### 第三步: 安装git

如下图所示

![](https://raw.githubusercontent.com/z-cursor/blog-pic/main/Post/git.png)

去[git官网](https://git-scm.com/install/windows)下载git的windows安装包, 之后的安装程序如下图

![](https://raw.githubusercontent.com/z-cursor/blog-pic/main/Post/git2.png)接下来在安装界面一路next最后点击finish

### 第四步: 确认git安装完毕

powershell中输入:

`git -v`

输出如`git version 2.47.1.windows.2`即可


### 第五步:安装 OpenClaw

![鼠标右键PowerShell](https://raw.githubusercontent.com/z-cursor/blog-pic/main/Post/powershell.png)
如图, 在搜索栏搜索powershell后, 右键或者从右边选项栏选择用管理员模式打开, 然后依次执行:

`Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser` 这个命令是避免powershell策略太严格导致的安装报错

`iwr -useb https://openclaw.ai/install.ps1 | iex` 这个命令是openclaw官方提供的下载命令


### 第六步:解决可能遇到的问题

以下是修改 npm 目录(作用是避免权限冲突):

`npm config set prefix "C:\npm"`

`npm config set cache "C:\npm-cache"`

以下是设置npm的安装仓库, 防止在国外仓库下载导致超时:

`npm config set registry https://registry.npmmirror.com`

以下命令是把新目录加到 PATH:

`[Environment]::SetEnvironmentVariable("Path",  $env:Path + ";C:\npm",  "User")` 

改完之后重新打开一个 PowerShell 窗口再试

另外, 所有的问题都可以考虑把报错信息复制下来粘贴到AI(比如qwen deepseek 智谱清言)去问它们, 之后按照它们的指导一步一步去做就可以了


### 第七步:验证和初始化

`openclaw --version`

`openclaw onboard --install-daemon `

后面的配置流程和 Mac 一致, 跟着向导走就行

## 方案二:WSL2 + Ubuntu

> 这是官方推荐的方式说白了就是在 Windows 里跑一个 Linux 环境, 体验和 Mac/Linux 基本一致

### 第一步:启用 WSL2
以管理员身份打开PowerShell（右键开始菜单 → Windows PowerShell (管理员)）, 然后执行:

`dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart`

`dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart`

`wsl --set-default-version 2`

执行完之后重启电脑, 这一步不能省

### 第二步:安装 Ubuntu

1. 打开 Microsoft Store
2. 搜索 `Ubuntu 22.04 LTS`（或`24.04 LTS`）点「获取」安装
3. 首次启动时设置一个用户名和密码（这是 Linux 里的, 随便设）

### 第三步:更新系统和装依赖

在powershell中输入wsl打开Ubuntu终端, 按照顺序执行:

1. `cd ~`
2. `sudo apt update && sudo apt upgrade -y`
3. `sudo apt install -y curl git wget build-essential` 

### 第四步:装 Node.js

`curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash`

`sudo apt install -y nodejs`

装完验证一下:
`node -v`

`npm -v `

### 第五步:装 OpenClaw

`curl -fsSL https://openclaw.ai/install.sh | bash `

### 第六步:验证和初始化

`openclaw --version `, 观察是否有在版本号(如2026.3.7), 之后的配置流程和 Mac 一致, 跟着向导走就行

### 第七步:从 Windows 访问

OpenClaw 跑在 WSL2 里, 但 Windows 浏览器可以直接访问, 在 WSL2 终端里启动:

`openclaw dashboard`

然后在 Windows 浏览器打开终端中显示的链接`http://localhost:18789`, 就能用了

# Windows 常见问题:

sharp 模块加载失败？

`npm cache clean --force`

`npm install -g openclaw@latest --force` 

Windows Defender 拦截了？
把这两个路径加到 Defender 排除项里:

`C:\Users\你的用户名\AppData\Roaming\npm`

`C:\Users\你的用户名\.openclaw`


# API 怎么配？

OpenClaw 本身不自带 AI 模型, 需要你接入一个

零配置方案:选 Qwen（新手首选）
前面说了, 配置向导里直接选 qwen, 授权登录就能用, 完全不需要折腾 API Key赶时间或者怕麻烦的直接选这个

缺点就是每天有免费问答次数限制, 超了会被提示如果后面觉得不够用, 再换成其他的自配方案

如果要选择其他大模型的API, 可以在相应的官网创建相对应的密钥,再将密钥配置到openclaw中,下面以Kimi为例

打开[kimi的官方开放平台](https://platform.moonshot.cn/console/api-keys),登录后,点击左侧菜单栏中的API keys,具体界面如下图:

![](https://raw.githubusercontent.com/z-cursor/blog-pic/main/Post/kimi2)

点击创建API keys, 也就是上图右上角的蓝色按键, 填写了API keys的名字后，会弹出来对应的密钥, 具体界面如下图:

![](https://raw.githubusercontent.com/z-cursor/blog-pic/main/Post/kimi3)

将它复制下来, 之后在终端中用`openclaw config`打开 openclaw配置界面, 在Model选项中选择Moonshot AI

在之后的选项中, 由于是在国内官网拿到的API keys, 所以选择.cn后缀的网站

之后选项中选择Paste API key now, 将先前复制的密钥粘贴在终端中, 全流程如图所示:

![](https://raw.githubusercontent.com/z-cursor/blog-pic/main/Post/20260310-API.png)

接下来两个选项是选择模型, 如果没有特殊要求, 直接回车,最后就配置好了模型

# 接入聊天工具

装好 OpenClaw 之后, 接入聊天工具才算真正”用起来”

飞书、Telegram、企业微信、QQ……这些平台的 Bot 配置方式各不相同, 需要的权限也不一样

好消息是官方文档把所有平台的配置步骤和权限列表都整理好了, 直接去看就行:

官方渠道配置文档:`https://docs.openclaw.ai/zh-CN/channels`

在浏览器打开上述网址, 里面有详细的创建步骤、权限清单、回调地址配置等, 照着做就行, 比我这里写得全多了

下面简单说一下大致流程, 让你心里有个数:
飞书（国内推荐）

大致流程:在飞书开放平台创建应用 → 配置事件订阅 → 开通权限（官方文档列了一大堆权限, 别漏了）→ 在 OpenClaw 里绑定

终端中运行`openclaw plugins install @m1heng-clawd/feishu`命令, 可以使openclaw安装相关插件

配置（替换成你自己的 App ID 和 Secret）

`openclaw config set channels.feishu '{"enabled":true, "appId":"cli_xxx", "appSecret":"xxx"}' --json` 

具体的 App ID 怎么拿、权限勾哪些, 直接看 官方文档 里的飞书部分, 写得很清楚

有问题的话可以去 OpenClaw 看看文档, 或者在社区里提问
